Research

Discover[i]

Nouraldin Jaber - Swen Jacobs - Milind Kulkarni - Roopsha Samanta

slides

Discover[i] is a framework built to tackle the challenging problem of parameterized verification and synthesis of distributed systems. The key challenges that Discover[i] is trying to address, along with our proposed solutions, are summarized below:

  • Distributed systems are made of complex building blocks! However, the good news is that most such components (e.g. consensus) have been extensively studied and proven correct. We then can abstract from such components and just reason about systems built using those components.

  • We need such systems to be correct for any number of processes: just because a system works for 3 processes, doesn’t mean it will work for 5!, Hence, we call for parameterized verification techniques.

  • Finally, Discover[i] aims to help programmers build distributed systems by synthesizing parts of such systems. The challenge here is to be able to synthesize parametrized completions that would work correctly regardless of the size of the distributed system.

Data Structure-Aware Heap Partitioning

Nouraldin Jaber - Milind Kulkarni

paper - slides

Heap or (program) partitioning is a key concept in many applications like computation offloading and region-based memory management.Fundamentally, such applications must generate code such that objects in the heap (and hence the code that operates on those objects) get partitioned depending on how those objects are used. Regardless of the intended application goal, the granularity at which the heap is partitioned is the key factor in partition quality.

Previous work suggested two main granularities:

  • Class-based: co-locate all objects with the same type in the same partition.
  • Allocation site–based: co-locate objects created at the same allocation site in the same partition.
Both approaches share a critical drawback: data structures that are used in different ways can share the same class, or the same allocation sites for internal objects, and hence are forced to be co-located despite their different usage patterns.

We introduce the notion of data structure–aware partitioning to allow different data structures to be placed in different partitions. Our strategy consists of an analysis that infers ownership properties between objects to identify data structures, and a code generation phase that encodes this ownership information into objects’ data types and allocation sites without changing the semantics of the code.
